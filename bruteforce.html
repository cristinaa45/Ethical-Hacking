Brute-Force Attacks

What is a Brute-Force Attack?

A brute-force attack is a method where every possible password combination is tried until the correct one is found.

This is not a specific tool — it is a strategy that tools like Hydra can perform.

Brute Force vs Dictionary Attack

Brute-force attack: tries every possible combination, which can take a long time but is very thorough.

Dictionary attack: uses a wordlist of common passwords, making it faster but less complete.

Example in a Penetration Test

During a penetration test, a tester might use Hydra to test login systems for weak passwords.

If a simple password like "admin123" is found quickly, it shows the system is vulnerable to credential attacks.

Defenses Against Brute-Force Attacks

  • Account lockouts after multiple failed attempts
  • Rate limiting to slow down login attempts
  • Multi-factor authentication (MFA) for extra security

What a Security Finding Looks Like

If a tester successfully guesses a password, it is reported as a security vulnerability.

The risk is unauthorized access to systems or data.

The recommendation would be:

  • Enforce stronger passwords
  • Enable MFA
  • Limit login attempts
